# AsciiDoc conventions for pages in this manifest:
#
#   - First line must be `= Page Title`. scripts/build-docs.sh extracts
#     this into the HTML <title> and <h1>; the body below is everything
#     after that first line.
#   - Section headings: `== Foo`, `=== Bar`. Don't use a second `=`-level
#     heading — the page-shell already supplies the H1.
#   - Code blocks: `[source,LANG]` followed by `----` fences. Use
#     `text` for plain literal blocks, `yaml`/`bash`/`rust` etc. for
#     syntax-highlighted output.
#   - Inline code: backticks.
#   - External links: `https://example.com[link text]`.
#   - Internal links to other built pages: `link:<flattened-name>.html[Title]`
#     where the flattened name is the source path with `/` → `-` and
#     `.adoc` stripped (so `reference/concepts.adoc` → `reference-concepts.html`).
#   - No `:doctype:`/`:toc:` attribute block needed; chrome and TOC come
#     from docs/templates/page-shell.html.
